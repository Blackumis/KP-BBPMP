A. Admin
1. Login ke sistem.
2. Mengisi informasi kegiatan
- Nama kegiatan
- Tanggal mulai dan selesai kegiatan
- Jam mulai dan selesai kegiatan
- Nomor surat kegiatan sekaligus sebagai dasar nomor surat sertifikat (urutanya nomor
sertifikat urut-absen/nomor surat kegiatan
- Upload Template/background sertifikat ( ukuran maksimal ditentukan untuk kertas A4)
3. Membuat form. (memilih field dari opsi yang tersedia)
● Nama (wajib)
● Unit Kerja (wajib)
● NIP
● Kabupaten/Kota Unit Kerja (wajib)
● Tanggal Lahir (wajib)
● Nomor Handphone (wajib)
● Pangkat/Golongan
● Jabatan
● E-mail (wajib)
● E-mail konfirmasi (wajib)
● e-Signature atau ttd elektronik peserta (wajib)
● Check box pernyataan (wajib) isi : “Menyetujui bahwa seluruh data yang tertera
sesuai dengan identitas asli dan penulisan dalam standar penulisan EYD v5”
● Form yang dibuat harus merujuk pada ID-Nama Kegiatan (wajib tidak tampil di
user)
4. Menyimpan, generate atau membuat llink form absensi.
5. Membagikan link absensi kepada peserta kegiatan.
B. User
7. Mengikuti kegiatan/webinar.
8. Membuka link absensi.
9. Mengisi data pada form absensi.
10. Klik tombol submit.(1. tombol submit akan muncul jika email dan email konfirmasi cocok,
checkbox persetujuan dicentang, tombol sumbit baru muncul. 2. diberi jeda waktu untuk antrian)
C. Sistem
11. Melakukan validasi data:
- Cek kelengkapan isian
- Cek format email dan data ( untuk data peserta absen model validasi datanya
bagaimana? Masih diperlukan tidak?)
- Cek absensi ganda untuk kegiatan yang sama (wajib)
12. Jika data tidak valid, sistem menampilkan pesan error.
13. Jika data valid, sistem menyimpan data absensi ke database.
14. Status data peserta diset sebagai “menunggu sertifikat”.
D. Admin
15. Login ke sistem.
16. Membuka halaman data absensi peserta.
17. Melakukan pengecekan dan perbaikan data (jika diperlukan).
18. Memilih template/background sertifikat.
19. Mengatur format penomoran sertifikat:
- Nomor urut
- Kode kegiatan
- Tanggal kegiatan
20. Menyimpan konfigurasi sertifikat.
21. Menjalankan proses generate sertifikat.
E. Sistem
22. Membuat nomor sertifikat otomatis untuk setiap peserta. (sesuai urutan absensi)
23. Memasukkan data peserta ke dalam template sertifikat.
24. Menghasilkan sertifikat dalam bentuk file PDF.
25. Menyimpan file sertifikat.
26. Mengirim sertifikat ke email peserta secara otomatis.
27. Status peserta diubah menjadi “sertifikat terkirim”.
F. Output
28. Peserta menerima sertifikat melalui email.
29. Admin dapat melihat riwayat sertifikat yang telah dikirim
